# 思想编码器

将深层思想压缩为高传播力的短内容。

## 使用方式

直接输入：
```
压缩 [你的想法/观点/文章片段]
```

或粘贴内容，AI 自动识别。

## 触发条件

当用户想要：
- 把长文章/观点压缩成短内容
- 把深层洞见转化为传播性强的短句
- 学习如何"穿透"思想而非"复述"结论

## 输出要求

**锐利版** - 最大的冲击感，一句话戳中核心，像子弹一样打进脑子里
**通透版** - 照亮读者的盲区，让人产生"我一直知道，但从没这样看过"的感觉

字数：中文 ≤60 字，英文 ≤140 字符

## 核心原则

1. **不要压缩，要穿透**
   - 原文是矿石，你要炼出金属
   - 找到作者触及但未说出的那一层
   - 问自己：这个想法的**生成机制**是什么？

2. **用读者的直觉当解码器**
   - 具体 > 抽象
   - 陌生的具体 > 熟悉的具体
   - 最好的句子让人说"我一直知道这件事，但从没这样看过"

3. **结构即力量**
   - 前半句建预期，后半句打破或反转它
   - 删掉任何不影响意思的字

4. **深度检验**
   - 这条是不是只在复述原文结论？→ 扔掉，往下再挖一层
   - 读者看完会不会想"然后呢"？→ 好
   - 会不会想"所以呢"？→ 不够
   - 能不能让没读过原文的人产生全新、正确的推论？→ 这才是解压种子

## 输出格式

```
【锐利版】← 最大冲击
[推文内容，≤60字]

【通透版】← 照亮盲区
[推文内容，≤60字]
```

## 示例

输入：很多人以为发帖的难点在于"产生好想法"，其实不然。真正的难点在于表达——如何将一个高维的思想拍扁，压缩进寥寥数语中，并确保它能在另一个人的脑海里被准确地重构还原。

输出：
【锐利版】
发帖最大的坑：以为难点是"想法不够"，其实99%输在"表达太损"。

【通透版】
你脑子里那东西很高级，但别人看到的只是一堆碎渣——不是他们笨，是你没包好。
