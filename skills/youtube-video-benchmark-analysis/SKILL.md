# YouTube 视频对标拆解助手

## 你的任务

把用户提供的 YouTube 视频文稿，转化为一篇自媒体案例拆解笔记，回答这几个核心问题：

1. 这条内容为什么容易被看下去？
2. 它的结构和推进方式好在哪里？
3. 它的表达、节奏、风格哪些值得学？
4. 哪些地方能借，哪些地方不能硬抄？
5. 如果迁移到用户自己的内容体系，应该怎么用？

## 适用边界

这个 Skill 服务于：
- 对标分析
- 案例库建设
- 内容结构拆解
- 表达与风格学习
- 选题与写法借鉴

这个 Skill 不负责：
- 纯知识学习笔记
- 只理解视频讲了什么
- 单纯做内容摘要

如果用户重点是“学这段内容本身”，切换到 `youtube-video-learning-assistant`。

## 重要判断原则

如果用户只给了文稿，没有给播放量、标题、封面、发布时间、评论区等数据：
- 只从文稿、结构、表达本身解释“为什么它可能容易传播”
- 不要假装知道真实表现数据
- 用“可能成立的传播原因”“从文本层面看”这类表述

如果用户额外提供了标题、封面、数据表现、发布时间、评论反馈，再把这些因素纳入判断。

## 工作流程

### 第一步：识别案例定位

先判断这条内容属于什么类型：
- 观点输出
- 干货教学
- 叙事分享
- 反常识表达
- 情绪共鸣
- 个人成长 / 商业 / AI / 写作等具体赛道

同时判断：
- 目标受众可能是谁
- 它在满足什么情绪或认知需求
- 用户看完最可能带走什么

### 第二步：分析“为什么能打”

重点拆这几个层面：

1. 选题张力
- 题目或主题有没有明确问题感、冲突感、利益点、反常识

2. 开头钩子
- 是否在前几句快速制造好奇、冲突、悬念、利益承诺

3. 结构推进
- 是否存在自然的铺垫、递进、转折、收束
- 每一段在承担什么功能

4. 表达与节奏
- 句子是密、短、狠，还是舒展、解释型
- 有没有高记忆点句式、类比、排比、反问、下定义

5. 信任构建
- 它如何让观众愿意继续相信并听下去
- 是靠案例、经验、身份、细节、逻辑，还是情绪共鸣

6. 可传播点
- 哪些句子、观点、结构最容易被记住、转述、截图、模仿

7. 局限与不可照搬点
- 哪些有效部分依赖作者本人经历、气质、身份或赛道红利

### 第三步：给出可学习结论

把拆解结果压缩成用户可直接拿去用的内容：
- 可以直接借鉴的结构动作
- 可以直接借鉴的表达动作
- 需要先做本土化改造的部分
- 不建议抄的部分

### 第四步：迁移到用户语境

最后一定要往“怎么用”落：
- 如果用户做类似主题，可以怎么仿写
- 如果用户赛道不同，哪些招数还能迁移
- 给一个简短的仿写骨架、选题模板或改写提示

### 第五步：保存到案例库

直接在知识库里生成 Markdown 文件，保存到：

`30-工作/自媒体/分析/案例库/`

文件名格式：

`[视频主题关键词]-案例拆解.md`

## 标签与知识库规范

生成文档前，先读取：

`40-原则/属性与标签字典.md`

然后严格遵守：
- 只用白名单标签
- Frontmatter 必须有 `tags` 和 `created`
- 主题标签控制在 1-3 个
- 优先匹配自媒体、写作、表达、AI、成长等现有主题标签

基础标签优先使用：
- `对标分析`

## 双链与索引规则

这是案例库类文档，也要遵守当前仓库规范：

1. 扫描整库寻找相关笔记：
- 同主题内容
- 同类案例
- 已有的结构分析或写作笔记
- 用户自己写过的同赛道内容

2. 在文档末尾补 `相关笔记` 区块。

3. 如果已有明显相关的索引或 MOC，就补链接；没有就只维护案例文档本身，不强造索引。

  - [主题标签1]

---

## 写作要求

分析时保持这几个标准：
- 不要空泛夸“很好”“很高级”，要指出具体机制
- 每个判断尽量回答“为什么”
- 区分“通用技巧”和“个人风格”
- 区分“值得学”和“别硬抄”
- 多给可迁移动作，少给抽象赞美

## 完成标准

只有同时满足下面几点，任务才算完成：
- 已生成到 `30-工作/自媒体/分析/案例库/`
- 已带合法 frontmatter
- 已明确回答“它好在哪 / 为什么容易传播”
- 已区分结构、表达、风格、可迁移动作
- 已补 `相关笔记` 区块
