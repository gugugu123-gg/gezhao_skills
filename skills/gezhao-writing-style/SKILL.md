## 第三步：根据模式改写

### 模式 A：口播版

**适用场景：** 视频脚本、音频播客、需要念出来的内容

**段落规则（核心）：**
- **每段 1-3 行，最多不超过 4 行**
- 一句一个意思，方便换气停顿
- 多用短句、断句

**示例：**
```
开学第二周的周六。

我已经走读整整两周了。

每天 6:20 起床，7:10 上地铁，7:35 进校门。

课少下午溜，课多晚上 7 点多到家。
```

---

### 模式 B：文章版

**适用场景：** 公众号、博客、知乎、需要阅读的文字内容

**段落规则（核心）：**
- **每段 3-5 行，最长不超过 6 行**
- 有完整的起承转合结构
- 保留口语化，但更有"文章感"
- 适合手机屏幕阅读，但不至于太碎

**示例：**
```
开学第二周的周六，我已经走读整整两周了。

每天 6:20 起床，7:10 上地铁，7:35 进校门。课少下午就能溜，课多晚上 7 点多到家。这节奏，跟高三有点像——只不过那时候早起还晚睡，现在起码睡得多了点。
```

## 两种模式的共同规则

### 规则 1：开头抓人

三种方式选其一：

**A. 问句开场**
```
你有没有想过，卷绩点正在偷走你的大学时间。
```

**B. 直接扔结论**
```
研究如何去「AI味」其实是一个伪命题。
```

**C. 场景引入**
```
前几天过年，晚上躺在床上突然想到一个事情。
```

### 规则 2：高频使用第一人称「我」

**改写时把第三人称改为第一人称：**
- ❌ "很多人觉得..." → ✅ "我觉得..."
- ❌ "大家会发现..." → ✅ "我发现..."
- ❌ "人们应该..." → ✅ "我应该..." 或 "你可以..."

### 规则 3：口语化表达

**常用口头禅（选择使用）：**
- 「说白了」
- 「其实」
- 「讲道理」
- 「那这个时候」
- "那么问题来了"
- "归根结底"

**网络/年轻用语（适度使用）：**
- 「卷」、「老登」
- 「可可又惜惜」
- 「懒猪」、「妖魔鬼怪」

### 规则 4：用类比解释抽象概念

**把抽象概念变成具体画面：**
- AI/方法论 → 做饭/菜谱
- 执行力 → 打猎
- 机制/系统 → 游戏

### 规则 5：小标题就是观点句

**不要用「第一章」「背景介绍」。**

**直接说观点：**
- 「卷绩点正在偷走你的大学时间」
- 「执行力的反面叫做预测未来」
- 「低成本启动」

### 规则 6：结尾金句或行动原则

**三种结尾方式：**

**A. 金句型**
```
你得先知道自己是谁，AI 才知道怎么像你。
```

**B. 原则型**
```
重建一套自己的系统，是我上大学学到的第一件事情。
```

**C. 行动型**
```
如果想到了一件事情，在简单考虑可行性之后，要在 5 秒内执行下去。
```

## 禁止清单（严格执行）

| 禁用内容 | 处理方式 |
|---------|---------|
| 「不是……而是……」 | **AI味重灾区，严格避免** | 直接说结论 |
| 「在当今社会/时代/背景下」 | 删除，直接切入 |
| 「我们必须/我们应该/我们要」 | 改为「我」或「你可以」 |
| 「众所周知/不可否认/毫无疑问」 | 删除 |
| 「笔者/本文/本研究」 | 改为「我/这篇文章」 |
| 「之于」「抑或」「乃」 | 改为「对于」「或者」「就是」 |
| 双重否定 | 改为直接肯定 |
| 形容词堆砌（「非常」「极其」等） | 用具体数字或例子替代 |
| "一方面……另一方面……" | 直接分点说 |

## 检查清单

**口播版额外检查：**
- [ ] 每段是否都不超过 4 行？
- [ ] 是否方便念出来、有停顿感？

**文章版额外检查：**
- [ ] 每段是否 3-6 行之间？
- [ ] 是否有完整的文章结构（起承转合）？
- [ ] 阅读起来是否流畅、不累？

**共同检查（两种模式都要）：**
- [ ] 是否用了第一人称「我」？
- [ ] 开头是否抓人（问句/结论/场景）？
- [ ] 小标题是否就是观点句？
- [ ] 结尾是否有金句或行动原则？
- [ ] 是否删除了所有禁用词？

---

## 关键提醒

1. **必须先询问用户选择模式**，不要假设。

2. **段落长度是区分两种模式的核心**：
   - 口播版：1-3 行（极短）
   - 文章版：3-5 行（适中）

3. **改变表达方式，不改变内容。** 保持原文的观点、例子和数据。

4. **如果不确定，参考范文。** 读取 `40-原则/我的写作风格.md` 中的参考范文链接。
